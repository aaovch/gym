# Правила для агента

В этом проекте данные по тренировкам хранятся в компактном JSON v4. Любые
правки данных считай рискованными: сохраняй историю тренировок пользователя,
держи канонические и статические файлы синхронизированными и проверяй результат
перед отчетом об успехе.

## Контракт правки данных

- Канонические файлы: `data/workouts.json`, `data/cycle-plan.json` и
  `data/training-theses.json`. Не правь `web/static/data/*.json` вручную и
  отдельно; сначала меняй канонические `data/*.json`, потом запускай
  синхронизацию.
- Сохраняй `version: 4`. Не переименовывай существующие id упражнений, логов,
  микроциклов и сессий, если пользователь явно не попросил миграцию.
- Сохраняй все существующие логи, комментарии, связи `microSessionId`,
  `anchor1rm`, `anchor1rmManual`, `exerciseProtocols` и `exerciseSessions`, если
  задача прямо не касается этих полей.
- При добавлении или изменении тренировки переиспользуй существующий id
  упражнения, если такое упражнение уже есть. Новое упражнение добавляй только
  когда подходящего существующего нет: со стабильным транслитерированным id,
  правильным `k` и релевантными тегами блока `b`.
- Для силовых подходов используй `weightKg` и `reps`. Для бега используй
  существующие поля беговых сетов. Для прыжков используй существующие поля
  прыжковых сетов. Не смешивай разные `kind` внутри одного блока.
- Даты должны быть календарными ISO-датами (`YYYY-MM-DD`). Не угадывай
  отсутствующую дату молча; если в команде нет даты, спроси пользователя.
- Делай минимально возможное изменение данных. Не делай широкое
  переформатирование, кроме штатного компактного JSON, который выдают
  существующие скрипты синхронизации и ремонта.

## Обязательная проверка

После любой правки данных запусти из корня репозитория:

```powershell
python .\scripts\sync_static_data.py
cd .\web
npm run check
npm run build
```

Если могли измениться связи циклов и логов, перед `check` и `build` запусти:

```powershell
cd .\web
npm run repair:data
```

В отчете точно укажи, какие канонические файлы изменились, были ли
синхронизированы статические копии и какие проверочные команды прошли или
упали.
